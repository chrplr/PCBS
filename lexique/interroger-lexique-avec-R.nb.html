Exploration de la distribution des frquences

Supprimons les mots trs rares, de frquence doccurrence infrieure 0.5 par million dans le corpus des livres :

lexique1 <- subset(lexique, freqlivres > 0.5)
lexique1$logfreq <- log10(lexique1$freqlivres)

Puis calculons lhistogramme des frquences, et affichons le sur un graphique avec des axes logarithmiques:

with(lexique1, {
  histdata <- hist(logfreq, plot=FALSE, nclass=50)
  plot(histdata$breaks[-1], histdata$count, log="y", type='h', lwd=10, lend=2, las=1, xlab='log10(freqlivres)', ylab='count')
  }
)

Recherche de mots

Supposons que vous vouliez extraire les lignes de la table lexique correspondant, par exemple, aux mots bateau, avion, maison, arbre.

Crez un bloc de code (Ctrl-Alt-I) et copiez-y le code suivant :

items <- c('bateau', 'avion', 'maison', 'arbre')

selection <- subset(lexique, ortho %in% items)

head(selection)

Excutez ce code (Ctrl-Shift-Enter). La variable selection doit contenir les lignes pertinentes. Vous pouvez inspecter son contenu en cliquant sur son nom dans longlet Environment situ dans la fentre en haut droite de RStudio.

Pour sauvegarder les rsultats obtenus dans un fichier, exctuez la commande suivante:

write_tsv(selection, 'selection.tsv')

Cela doit crer un fichier selection.tsv dans le rpertoire du projet (lexique).

Notez que les fichiers ayant lextension tsv (tab-separated-values) peuvent tre ouverts avec Excel, ou OpenOffice Calc, ou mme avec nimporte quel diteur de texte. Note: le package readr de R fournit aussi des fonctions write_excel_csv et write_excel_csv2 qui peuvent intresser certains.

Si vous avez une liste de mots plus longue, il serait fastidieux dcrire la ligne items <- .... Plus simplement vous pouvez utiliser:

items = scan(what='characters')
bonjour

Read 1 item

Et coller la liste de mots. Pour finir la liste, entrez une ligne vide (en appuyant deux fois successivement sur la touche Entre), puis, nouveau:

selection <- subset(lexique, ortho %in% items)

Comme la fonction scan permet aussi de lire la liste dans un fichier externe, si vous avez une liste de mots dans fichier liste.txt, vous pouvez extraire les lignes de lexique qui contiennent spcifiquement ces mots avec le code suivant:

liste <- scan('liste.txt', what='characters')
Read 2 items
selection <- subset(lexique, ortho %in% liste)

Slection par critres

Supposons que vous vouliez slectionner tous les noms de 5 lettres, singuliers, de frquence lexicale (films) comprise entre 10 et 100. Voici la ligne magique:

selection = subset(lexique, cgram=='NOM' & nombre != 'p' & nblettres==5 & freqlivres > 10 & freqlivres < 100)
head(selection)

Le symbole & signifie et. Lexpression nombre != p signitif que la valeur dans la colonne nombre ne doit pas tre p.

Slection par pattern

Les expressions rgulires

Les expressions rulires, ou regex, sont des patterns qui permettent de rechercher des mots ayant certaines proprits. Par exemple nimporte a.b dsigne un mot contenant un a et un b spars par une lettre quelconque. Voici dautre exemples:

  • ^maison$ : recherche le mot maison exactement
  • ^anti : recherche tous les mots commenant par anti
  • ^(jour|nuit|matin|soir)$ : jour ou nuit ou matin ou soir (permet de rechercher une liste de mots)
  • ion : recherche les mots qui contiennent la chaine ion dans nimporte quelle position
  • ion$ : mots se terminant par ion
  • ^pr : mots commenant par pr
  • ^p..r$ : mots de quatre lettres commenant par p, finisant pas r
  • ^p.*r$ : mots commencant par p et finissant par r
  • [aeiou][aeiou] : mots contenant 2 voyelles successives
  • ^[aeiou] : mots commenant par une voyelle
  • ^[^aeriou] : mots ne commenant pas par une voyelle

Il existe de nombreux tutoriaux sur les regex sur le web, notamment http://regextutorials.com/intro.html

Une expression rgulire dcrit un automate de transitions tats finis. Le site https://regexper.com/ vous permet de visualiser lautomate associ une regex. Par exemple [ptk].*[aiou][aiou].?ion$ correspond lautomate fini:

Recherches dans R avec grepl

R permet deffectuer des recherches par pattern grce la fonction grepl. La syntaxe est grepl(regex, variable) pour rechercher les lignes o la variable matche la regex (Voir la doc R de grepl.

Cette fonction permet de localiser les lignes qui matchent une expression, ou bien, en la niant avec le signe !, de supprimer des lignes qui matchent un pattern.

Voici quelques exemples:

  • Pour obtenir tous les mots qui finissent par tion :
lexique %>% filter(grepl("tion$", ortho)) -> selection2
head(selection2)

Encore une fois, vous pouvez sauvegarder ces rsultats avec:

write_tsv(selection2, 'mots-en-tion.tsv')
  • Pour lister tous les mots contenant un cluster de consonnes plosives, mais pas debut de mot:
lexique %>% filter(grepl('.[ptkbdg][ptkbdg]', phon)) -> selection3
head(selection3)

  • Loprateur filterpeut tre appeler plusieurs fois pour affiner progressivement la recherche.

    Par exemple, pour obtenir tous les mots de 8 lettres qui ne finissent pas ent:

lexique %>% filter(nblettres == 8) %>% filter(!grepl("ent$", ortho)) -> selection4
head(selection4)

Chargement dautres tables dOpenLexicon

Le projet OpenLexicon comprend de nombreuses tables lexicales (voir https://chrplr.github.io/openlexicon/datasets-info/).

Pour accder ces tables, vous pouvez utiliser la fonction fetch_datatset dfinie dans https://raw.githubusercontent.com/chrplr/openlexicon/master/datasets-info/fetch_datasets.R.

Elle tlcharge automatiquement les bases de donnes dfinies dans les fichiers .json lists dans https://raw.githubusercontent.com/chrplr/openlexicon/master/datasets-info/_json.

Par exemple, pour charger la table SUBTLEX-US:

```r
source('https://raw.githubusercontent.com/chrplr/openlexicon/master/datasets-info/fetch_datasets.R')
subtlexus <- readRDS(fetch_dataset('SUBTLEX-US', format='rds')$datatables[[1]])
```
